# DataAgent 项目结构分析说明

## 📋 项目概况

**DataAgent** 是一个基于 Spring AI Alibaba Graph 构建的**企业级智能数据分析 Agent**，它不仅是一个 Text-to-SQL 工具，而是一个能够执行 Python 深度分析、生成多维度图表报告的 AI 数据分析师。

## 🏗️ 技术架构

### 后端技术栈

- **核心框架**: Spring Boot 3.4.8+ / Java 17+
- **AI 框架**: Spring AI Alibaba 1.1.0.0 + Spring AI 1.1.0
- **工作流引擎**: Spring AI Alibaba StateGraph
- **数据持久层**: MyBatis 3.0.4 + MySQL 5.7+
- **连接池**: Druid 1.2.22
- **向量存储**: Elasticsearch (可扩展其他向量数据库)
- **文档处理**: Apache Tika, EasyExcel 3.3.4
- **代码执行**: Docker Java Client (Python 执行器)
- **协议支持**: MCP (Model Context Protocol)

### 前端技术栈

- **框架**: Vue 3 (Composition API) + Vite 5.0
- **UI 组件**: Element Plus 2.9.11
- **路由**: Vue Router 4
- **图表**: ECharts 6.0
- **Markdown**: markdown-it 12.0.6 + marked 12.0.2
- **代码高亮**: highlight.js 11.11.1
- **类型检查**: TypeScript 5.0

## ✨ 核心特性

### 1. 智能数据分析
- 基于 StateGraph 的 Text-to-SQL 转换
- 支持复杂多表查询和多轮对话
- 智能意图识别和查询重写

### 2. Python 深度分析
- 内置 Docker/Local Python 执行器
- 自动生成并执行 Python 代码
- 支持统计分析和机器学习预测

### 3. 智能报告生成
- 自动生成 HTML/Markdown 报告
- 集成 ECharts 多维度图表
- 支持流式输出和实时渲染

### 4. Human-in-the-loop 机制
- 计划生成阶段的人工审核
- 支持批准/拒绝/修改计划
- 通过 `CompiledGraph.interruptBefore()` 实现暂停/恢复

### 5. RAG 检索增强
- 向量数据库集成(默认 Elasticsearch)
- 支持业务知识、语义模型、Agent 知识检索
- 混合检索策略(向量+关键词)

### 6. 多模型调度
- 运行时动态切换 LLM 和 Embedding 模型
- 全面兼容 OpenAI API 规范
- 支持 Qwen、Deepseek 等主流模型

### 7. MCP 服务器支持
- 原生支持 MCP 协议
- 可作为 Tool Server 集成到 Claude Desktop
- 提供 NL2SQL 和 Agent 管理工具

## 🔄 工作流设计 (StateGraph Nodes)

项目使用 16 个核心节点构建智能分析工作流：

| 节点名称 | 职责 | 关键功能 |
|---------|------|---------|
| **IntentRecognitionNode** | 意图识别 | 判断用户问题类型和分析需求 |
| **QueryEnhanceNode** | 查询增强 | 优化和重写用户查询 |
| **EvidenceRecallNode** | 证据召回 | 从知识库检索相关上下文 |
| **SchemaRecallNode** | Schema 召回 | 获取数据库表结构信息 |
| **TableRelationNode** | 表关系分析 | 识别多表关联关系 |
| **SemanticConsistencyNode** | 语义一致性检查 | 确保查询语义正确 |
| **PlannerNode** | 计划生成 | 生成分析执行计划 |
| **HumanFeedbackNode** | 人工反馈 | 人工审核计划 |
| **PlanExecutorNode** | 计划执行 | 编排任务执行流程 |
| **SqlGenerateNode** | SQL 生成 | 生成 SQL 查询语句 |
| **SqlExecuteNode** | SQL 执行 | 执行 SQL 并获取结果 |
| **PythonGenerateNode** | Python 生成 | 生成 Python 分析代码 |
| **PythonExecuteNode** | Python 执行 | 在 Docker/Local 执行代码 |
| **PythonAnalyzeNode** | Python 分析 | 分析执行结果 |
| **ReportGeneratorNode** | 报告生成 | 生成最终分析报告 |
| **FeasibilityAssessmentNode** | 可行性评估 | 评估任务可执行性 |

## 🗄️ 数据库设计

项目包含 11 张核心业务表：

| 表名 | 用途 | 关键字段 |
|------|------|---------|
| `agent` | 智能体配置 | name, description, api_key, status, human_review_enabled |
| `business_knowledge` | 业务知识库 | business_term, synonyms, is_recall, embedding_status |
| `semantic_model` | 语义模型 | business_name, synonyms, business_description |
| `agent_knowledge` | Agent 知识源 | type(DOCUMENT/QA/FAQ), content, file_path, splitter_type |
| `datasource` | 数据源配置 | type, host, port, connection_url, status |
| `logical_relation` | 逻辑外键 | source_table, target_table, relation_type |
| `agent_datasource` | Agent-数据源关联 | agent_id, datasource_id, is_active |
| `agent_preset_question` | 预设问题 | question, sort_order |
| `chat_session` | 会话管理 | title, status, is_pinned |
| `chat_message` | 消息记录 | role, content, message_type, metadata |
| `model_config` | 模型配置 | type(CHAT/EMBEDDING), api_key, is_active |

## 🚀 核心流程

### 典型 NL2SQL 流程

```
用户提问 
  ↓
IntentRecognitionNode (意图识别)
  ↓
EvidenceRecallNode (知识检索)
  ↓
SchemaRecallNode (Schema获取)
  ↓
PlannerNode (生成计划)
  ↓
[HumanFeedbackNode] (可选:人工审核)
  ↓
SqlGenerateNode (生成SQL)
  ↓
SqlExecuteNode (执行查询)
  ↓
ReportGeneratorNode (生成报告)
  ↓
返回结果 (SSE流式输出)
```

### Python 深度分析流程

```
基础SQL结果
  ↓
PythonGenerateNode (生成Python代码)
  ↓
PythonExecuteNode (Docker/Local执行)
  ↓
PythonAnalyzeNode (结果分析)
  ↓
ReportGeneratorNode (整合报告)
```

## 🔧 配置要点

### 后端配置 (`application.yml`)

- **服务端口**: `8065`
- **向量检索**: TopK=8, 相似度阈值=0.4
- **Python 执行器**: 默认 `local` (生产建议 `docker`)
- **LLM 服务类型**: `stream` (流式输出)
- **SQL 重试次数**: 10次

### 前端配置 (`package.json`)

- **开发服务器**: `127.0.0.1:3000`
- **代理配置**: `/api` 和 `/nl2sql` 代理到后端 `8065`

## 🔌 扩展开发

### 支持的扩展点

1. **向量数据库**: 通过实现 `VectorStore` 接口支持新的向量数据库
2. **AI 模型**: 兼容 OpenAI API 的模型可直接接入
3. **混合检索策略**: 继承 `AbstractHybridRetrievalStrategy` 自定义检索逻辑
4. **代码执行器**: 实现 `CodeExecutor` 接口支持新的执行环境
5. **文本分割器**: 支持 token、recursive、sentence、semantic 四种分割策略

## 🛠️ 开发环境搭建

### 1. 环境准备

- JDK 17+
- MySQL 5.7+
- Node.js 16+
- Docker (可选，用于 Python 执行)

### 2. 数据库初始化

```bash
mysql -u root -p < data-agent-management/src/main/resources/sql/schema.sql
```

### 3. 启动后端

```bash
cd data-agent-management
./mvnw spring-boot:run
```

### 4. 启动前端

```bash
cd data-agent-frontend
npm install && npm run dev
```

### 5. 访问系统

- 前端界面: http://localhost:3000
- 后端 API: http://localhost:8065
